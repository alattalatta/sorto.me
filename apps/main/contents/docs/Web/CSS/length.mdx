CSS **`<length>`** 자료형은 길이를 나타내며, 다양한 CSS 속성의 값으로 사용됩니다. [`width`](https://developer.mozilla.org/en-US/docs/Web/CSS/width), [`height`](https://developer.mozilla.org/en-US/docs/Web/CSS/height), [`padding`](https://developer.mozilla.org/en-US/docs/Web/CSS/padding)처럼 너비와 높이, [`margin`](./margin)과 [`inset`](./inset)처럼 거리, 그리고 [`font-size`](https://developer.mozilla.org/en-US/docs/Web/CSS/font-size)처럼 크기 자체를 나타낼 때 모두 `<length>`가 쓰입니다.

> [note] 일부 CSS 속성에서는 `<length>`를 받을 수 있는 곳에 [`<percentage>`](https://developer.mozilla.org/en-US/docs/Web/CSS/percentage)도 사용할 수 있지만, `<percentage>`는 `<length>`와 다른 별도의 자료형입니다. [`<length-percentage>`](https://developer.mozilla.org/en-US/docs/Web/CSS/length-percentage)를 참고하세요.

# 구문

`<length>` 자료형은 [`<number>`](./number)와 그 뒤의 단위로 구성됩니다. 다른 모든 CSS의 수치와 마찬가지로 수와 단위 사이에 공백이 있어선 안됩니다. 수가 0일 땐 단위를 붙이지 않아도 됩니다.

> [note] 일부 속성은 음의 `<length>`를 허용하지만, 모든 `<length>`를 사용하는 속성이 음수를 받을 수 있는 것은 아닙니다.

## 단위

### 상대 길이 단위

상대 길이는 어떤 다른 길이를 기준으로 삼아 측정한 값입니다. 단위에 따라 기준 길이가 다른데, 특정 문자의 너비/높이나 줄 간격, 뷰포트 크기 등을 사용합니다.

#### 글꼴 상대 길이

글꼴 상대 길이는 현재 요소에 적용된 글꼴의 특성 값이나 어떤 특정 문자의 크기를 기준으로 `<length>` 값을 정의합니다.

> [note] 글꼴 상대 길이 단위, 특히 [`em`](#term-em)과 [`rem`](#term-rem)은 사용자가 글꼴 크기를 변경하더라도 페이지의 수직 흐름을 유지하는 유연한 레이아웃을 만들 때 자주 사용합니다.

- : `cap` <Experimental />
  - 요소 글꼴의 대문자 높이(cap height)를 나타냅니다.
- : `ch`
  - 요소 글꼴에서 문자 "0"(숫자 영, Unicode `U+0030`)의 _진행 척도_(advance measure)를 나타냅니다. [진행 척도](https://drafts.csswg.org/css-values/#length-advance-measure)란 요소의 인라인 방향에 따라 어떤 문자가 차지하는 너비(가로쓰기) 또는 높이(세로쓰기)를 말합니다.

    문자 "0"의 진행 척도를 알아내기가 (사실상) 불가능한 경우 너비로는 `0.5em`, 높이로는 `1em`으로 취급합니다.

- : `em`
  - 요소 [`font-size` 속성](https://developer.mozilla.org/en-US/docs/Web/CSS/font-size)의 [계산 값](https://developer.mozilla.org/en-US/docs/Web/CSS/computed_value)을 나타냅니다. `font-size` 속성에 `em` 단위를 사용하는 경우 요소가 상속한 `font-size` 값을 나타냅니다.
- : `ex`
  - 요소 글꼴에서 문자 "x"의 높이를 나타냅니다. 글꼴이 문자 "x"를 포함한다면 보통 다른 소문자의 높이와도 같으며, 많은 글꼴에서 `1ex` ≈ `0.5em`입니다.
- : `ic`
  - 요소 글꼴에서 문자 "水"(CJK 물 한자, Unicode `U+6C34`)의 _진행 척도_(advance measure)를 나타냅니다. 진행 척도의 정의는 [`ch` 단위의 설명](#term-ch)를 참고하세요.
- : `lh` <Experimental />
  - 요소 [`line-height` 속성](https://developer.mozilla.org/en-US/docs/Web/CSS/line-height)의 [계산 값](https://developer.mozilla.org/en-US/docs/Web/CSS/computed_value)을 나타냅니다. `line-height`의 값이 `normal`이면 절대 길이로 변환해 사용합니다.
- : `rem`
  - 루트 요소(보통 [`<html>`](/docs/Web/HTML/Element/html)) [`font-size` 속성](https://developer.mozilla.org/en-US/docs/Web/CSS/font-size)의 [계산 값](https://developer.mozilla.org/en-US/docs/Web/CSS/computed_value)을 나타냅니다. 루트 요소의 `font-size` 속성에 `rem` 단위를 사용하는 경우 `font-size`의 [초기 값](https://developer.mozilla.org/en-US/docs/Web/CSS/initial_value)을 나타냅니다. 대부분의 브라우저에서 초기 값은 `16px`이지만, 사용자 설정에 따라 달라질 수 있습니다.
- : `rlh` <Experimental />
  - 루트 요소(보통 [`<html>`](/docs/Web/HTML/Element/html)) [`line-height` 속성](https://developer.mozilla.org/en-US/docs/Web/CSS/line-height)의 [계산 값](https://developer.mozilla.org/en-US/docs/Web/CSS/computed_value)을 나타냅니다. `line-height`의 값이 `normal`이면 절대 길이로 변환해 사용합니다. 루트 요소의 `font-size` 또는 `line-height` 속성에 `rlh` 단위를 사용하는 경우 각 속성의 [초기 값](https://developer.mozilla.org/en-US/docs/Web/CSS/initial_value)을 나타냅니다.

#### 뷰포트 백분율 길이

뷰포트 백분율 길이는 [뷰포트](https://developer.mozilla.org/en-US/docs/Glossary/Viewport), 즉 현재 문서에서 볼 수 있는 부분의 크기를 기준으로 `<length>` 값을 정의합니다. [`@page`](https://developer.mozilla.org/en-US/docs/Web/CSS/@page) 선언 블록에는 뷰포트 백분율 길이를 사용할 수 없습니다.

- : `vh`
  - 뷰포트 최초 [컨테이닝 블록](https://developer.mozilla.org/en-US/docs/Web/CSS/Containing_block)의 높이 1%와 같습니다.
- : `vw`
  - 뷰포트 최초 [컨테이닝 블록](https://developer.mozilla.org/en-US/docs/Web/CSS/Containing_block)의 너비 1%와 같습니다.
- : `vi` <Experimental />
  - 루트 요소의 [인라인 축](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_Logical_Properties#block_vs._inline) 방향에 대한 뷰포트 최초 [컨테이닝 블록](https://developer.mozilla.org/en-US/docs/Web/CSS/Containing_block)의 크기 1%와 같습니다.
- : `vb` <Experimental />
  - 루트 요소의 [블록 축](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_Logical_Properties#block_vs._inline) 방향에 대한 뷰포트 최초 [컨테이닝 블록](https://developer.mozilla.org/en-US/docs/Web/CSS/Containing_block)의 크기 1%와 같습니다.
- : `vmin`
  - `vw`와 `vh` 중 작은 값과 같습니다.
- : `vmax`
  - `vw`와 `vh` 중 큰 값과 같습니다.

### 절대 길이 단위

절대 길이는 출력 매체의 물리적 속성에 기반한 길이를 나타냅니다. 먼저 어떤 단위 하나를 출력 매체의 물리 단위와 연관지어 기준으로 삼고, 다른 단위는 기준 단위에 상대적으로 정의하는 방식입니다. 화면과 같은 저해상도 장치와, 프린터와 같은 고해상도 장치에서의 기준 단위가 서로 다릅니다.

저해상도 장치에서는 `px` 단위가 물리적인 픽셀 하나의 크기에 대응하며, 다른 단위는 픽셀에 상대적으로 정의됩니다. 예컨대, `1in`은 `96px`이자 `72pt`입니다. 이 방식의 부작용은 인치(`in`), 센티미터(`cm`), 밀리미터(`mm`) 단위를 사용한 길이가 실제 물리적 인치, 센티미터, 밀리미터 길이와 다를 수 있다는 점입니다.

고해상도 장치에서는 인치(`in`), 센티미터(`cm`), 밀리미터(`mm`) 단위가 물리적 길이와 일치하며, `px`이 이에 상대적으로 정의됩니다. (1/96인치)

> [note] 많은 사용자들이 사용자 에이전트의 기본 글씨 크기를 키워서 사용합니다. 절대 길이 단위는 사용자 설정의 영향을 받지 않는 고정된 단위므로 접근성을 저해할 수 있습니다. 그러므로, [`font-size`](https://developer.mozilla.org/en-US/docs/Web/CSS/font-size)를 정할 땐 [`em`](#term-em)이나 [`rem`](#term-rem)처럼 상대 길이 단위를 사용하세요.

- : `px`
  - 픽셀 한 개입니다. 화면에서는 전통적으로 장치의 픽셀 하나를 나타냈습니다. 하지만 프린터와 고해상도 화면에서는 하나의 CSS 픽셀이 다수의 장치 픽셀에 대응합니다. 이때 `1px`은 `1in`의 1/96으로 정의됩니다.
- : `cm`
  - 1 센티미터입니다. `1cm` = `(96 / 2.54)px`입니다.
- : `mm`
  - 1 밀리미터입니다. `1mm` = `(1 / 10)cm`입니다.
- : `Q` <Experimental />
  - 0.25 밀리미터입니다. `1Q` = `(1 / 40)cm`입니다.
- : `in`
  - 1 인치입니다. `1in` = `2.54cm` = `96px`입니다.
- : `pc`
  - 1 피카입니다. `1pc` = `12pt` = `(1 / 6)in`입니다.
- : `pt`
  - 1 포인트입니다. `1pt` = `(1 / 72)in`입니다.

# 보간

애니메이션에서 `<length>` 값은 부동소수점 실수로서 보간됩니다. 보간에는 [계산 값](https://developer.mozilla.org/en-US/docs/Web/CSS/computed_value)을 사용합니다. 보간 속도는 애니메이션이 사용하는 [타이밍 함수](https://developer.mozilla.org/en-US/docs/Web/CSS/easing-function)가 결정합니다.
