CSS **`<named-color>`** 자료형은 `red`, `blue`, `black`, `lightseagreen` 등 사전 정의된 색상 키워드입니다. 구문적으로는 [`<ident>`](https://developer.mozilla.org/en-US/docs/Web/CSS/ident)에 속합니다.

[`<color>`](.)를 사용할 수 있는 모든 곳에는 `<named-color>`를 사용할 수 있습니다.

# 구문

```css
color: red;
color: orange;
color: tan;
color: rebeccapurple;
color: transparent;
```

## 값

`<named-color>`는 표준 색상, `transparent`, `currentcolor`로 구성됩니다.

### `transparent`

`transparent` 키워드는 완전히 투명한 색상을 나타냅니다. 즉 색상을 적용한 항목 뒤의 배경이 완전히 보입니다. 기술적으로 `transparent`는 `rgb(0 0 0 / 0%)`의 단축어입니다.

불투명한 색에서 투명한 색으로 나아가는 [`<gradient>`](https://developer.mozilla.org/en-US/docs/Web/CSS/gradient) 등에서 갑자기 검은색이 나타나는 걸 방지하기 위해, 현재의 CSS 명세에서는 `transparent`를 [프리멀티플라이드 알파](https://www.w3.org/TR/css-color-4/#interpolation-alpha)(premultiplied alpha) 색 공간에서 계산해야 한다고 명시하고 있습니다. 그러나 오래된 브라우저는 `transparent`를 처리할 때 0의 알파 값을 가진 검정색으로 처리할 수 있으므로 주의해야 합니다.

# 설명

모든 키워드는 [sRGB 색 공간](https://ko.wikipedia.org/wiki/SRGB) 내의 색을 가리키며 투명하지 않습니다. 색 이름들은 연상되는 색상을 가리키긴 하지만, 모든 이름들은 엄격한 근거 없이 임의로 지어진 것입니다.

많은 키워드가 [X11](https://ko.wikipedia.org/wiki/X_%EC%9C%88%EB%8F%84_%EC%8B%9C%EC%8A%A4%ED%85%9C)에서 가져온 것이긴 하지만 실제 RGB 값은 X11 시스템에서의 색상과 다를 수 있습니다. 제조사들이 X11 색상을 하드웨어에 맞게 조정하기 때문입니다.
