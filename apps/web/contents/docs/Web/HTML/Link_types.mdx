HTML에서의 **링크 유형**은 문서와 [`<a>`](./Element/a), [`<area>`](./Element/area), [`<form>`](./Element/form), [`<link>`](./Element/link) 요소로 링크된 리소스 사이의 관계를 나타냅니다.

    <tr>
      <td>`alternate`</td>
      <td>
        - [`rel="stylesheet"`](./Element/link#term-rel)인 `<link>` 요소인 경우, [대체 스타일 시트](https://developer.mozilla.org/en-US/docs/Web/CSS/Alternative_style_sheets)를 정의하는 링크입니다. 이 경우 [`title` 특성](./Element/link#term-title)도 지정해야 합니다.
        - [`type`](./Element/link#term-type)이 `application/rss+xml` 또는 `application/atom+xml`인 경우 신디케이션 피드를 정의하는 링크입니다. 페이지 내에서 처음으로 나타난 링크가 기본 피드를 나타냅니다.
        - 이외의 경우에는 링크가 대체 페이지를 정의합니다.
          - [`media`](./Element/link#term-media) 특성을 지정한 경우, 다른 장치(휴대기기 등)를 위한 페이지
          - [`hreflang`](./Element/link#term-hreflang) 특성을 지정한 경우, 다른 언어로 된 페이지
          - [`type`](./Element/link#term-type) 특성을 지정한 경우, 다른 형식(PDF 등)의 페이지
          - 위의 조합
      </td>
      <td>
        [`<a>`](./Element/a), [`<area>`](./Element/area), [`<link>`](./Element/link)
      </td>

        > [note]
        > `mailto:` 하이퍼링크에 사용할 수도 있지만, 공개 페이지에서 `mailto:`를 사용할 경우 크롤러 로봇들이 메일 주소를 수집한 후 막대한 양의 스팸 메일을 전송할 수 있습니다. 이런 경우에는 연락 양식 페이지로 안내하는 편이 더 좋습니다.
      </td>
      <td>
        [`<a>`](./Element/a), [`<area>`](./Element/area), [`<link>`](./Element/link)
      </td>
    </tr>

        여러 글을 정리해 모아놓은 페이지에서 각각의 글로 향하는 링크를 나타낼 때 사용할 수 있습니다.
      </td>

      </td>
      <td>

        - `<link>` 요소인 경우, 전체 페이지에 대한 도움말을 제공하는 리소스의 링크를 정의합니다.

        사용자 인터페이스에서 현재 페이지를 나타낼 리소스를 정의합니다. 보통 시/청각적 아이콘의 형태입니다. 브라우저에서는 *파비콘*이라는 이름으로도 불립니다.

        복수의 `<link rel="icon">` 요소가 존재하면, 브라우저는 각각의 [`media`](./Element/link#term-media), [`type`](./Element/link#term-type), [`sizes`](./Element/link#term-sizes) 특성 값을 사용해 제일 적합한 아이콘을 선택합니다. 여러 아이콘이 동등하게 적합하다면 제일 마지막 아이콘을 선택합니다. 세 특성의 값에 따라 가장 적합한 아이콘을 선택했으나, 지원하지 않는 이미지 형식 등의 이유로 사용 불가능한 경우에는 다음으로 적합한 아이콘을 선택합니다.

      <td>`noopener`</td>
      <td>
        새로 열리는 브라우징 맥락의 [`window.opener`](https://developer.mozilla.org/en-US/docs/Web/API/Window/opener) 속성을 `null`로 설정해서, 현재 문서로 접근하지 못하도록 합니다.

        신뢰할 수 없는 링크를 열 때, `Referer` HTTP 헤더는 제공하면서 목적지 페이지가 현재 문서를 조작하는 것을 막을 수 있습니다.

        `noopener`를 사용할 경우, `_top`, `_self`, `_parent`를 제외한 `target`은 모두 `_blank`로 취급되어 새 창/탭을 엽니다.
      </td>
      <td>
