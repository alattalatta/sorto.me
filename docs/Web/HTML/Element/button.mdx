**HTML `<button>` 요소**는 클릭 가능한 버튼을 나타냅니다. 양식의 제출 버튼 외에도 접근성을 준수하는 표준 버튼이 필요하면 문서 어디에서나 사용할 수 있습니다.

버튼의 기본 스타일은 사용자 에이전트가 구동 중인 플랫폼의 버튼과 유사한 형태를 가지지만, CSS를 사용하면 원하는 대로 바꿀 수 있습니다.

<HTMLDemo height={100}>

```html
<button type="button">
  즐겨찾기에 추가
</button>
```

```css
button {
  background: cornflowerblue;
  border: none;
  border-radius: 4px;
  box-shadow: inset 2px 2px 3px rgba(255, 255, 255, .6),
              inset -2px -2px 3px rgba(0, 0, 0, .6);
  color: #fff;
  font-size: 1em;
  padding: 8px 20px;
}

button:hover {
  background: dodgerblue;
}

button:active {
  box-shadow: inset -2px -2px 3px rgba(255, 255, 255, .6),
              inset 2px 2px 3px rgba(0, 0, 0, .6);
}
```

</HTMLDemo>

# 특성

[전역 특성](/docs/Web/HTML/Global_attributes)을 포함합니다.

<dl>
<Term>disabled</Term>
<dd>지정하면 사용자의 상호작용을 비활성화하는, 즉 버튼의 클릭과 포커스 획득을 막는 불리언 특성입니다.</dd>

<Term>form</Term>
<dd>

이 버튼을 연결할 _양식 소유자_([`<form>`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form) 요소)의 ID를 지정합니다. 같은 문서 상에 존재하는 `<form>`의 `id` 특성을 지정할 수 있습니다. 이 특성이 존재하지 않으면 `<button>`의 양식 소유자는 가장 가까운 조상 `<form>`이 됩니다. (존재할 경우)

`form` 특성을 사용하면 `<button>`이 `<form>`의 자손이 아니어도 연결할 수 있습니다. 조상에 `<form>`이 있는데 `form` 특성을 지정한 경우 특성이 가리키는 `<form>`이 우선합니다.

</dd>

<Term>formaction</Term>
<dd>

버튼을 클릭했을 때의 데이터를 제출할 URL을 지정합니다. 버튼의 *양식 소유자*에 지정한 [`action`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#attr-action) 특성보다 우선합니다. 양식 소유자가 없으면 아무 효과도 없습니다.

</dd>

<Term>formenctype</Term>
<dd>

버튼이 (*양식 소유자*가 존재하고, `type="button"`을 지정하지 않은) 제출용 버튼일 때, 제출할 데이터를 인코딩하는 방법을 지정합니다. 가능한 값은 다음과 같습니다.

- `application/x-www-form-urlencoded`: 기본값입니다.
- `multipart/form-data`: [`type`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input#attr-type)이 `file`인 [`<input>`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input)을 제출할 때 사용합니다.
- `text/plain`: 디버깅 용도로 사용하는 값입니다. 실제 사용자가 제출하는 데이터의 인코딩에는 사용하지 말아야 합니다.

버튼의 양식 소유자에 지정한 [`enctype`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#attr-enctype) 특성보다 우선합니다.

</dd>

<Term>formmethod</Term>
<dd>

버튼이 (*양식 소유자*가 존재하고, `type="button"`을 지정하지 않은) 제출용 버튼일 때, 제출할 데이터를 전송할 [HTTP 메서드](https://developer.mozilla.org/en-US/docs/Web/HTTP/Methods)를 지정합니다. 가능한 값은 다음과 같습니다.

- `post`: 양식의 데이터를 HTTP 요청의 본문에 넣어서 서버로 전송합니다. 로그인 키처럼 공개해선 안되는 정보를 포함해야 할 때 사용하세요.
- `get`: 양식의 데이터를 `action` URL 뒤의 쿼리 문자열로 붙여서 서버로 전송합니다. 검색 창처럼 [양식이 유발하는 부수효과가 없을 때](https://developer.mozilla.org/ko/docs/Glossary/Idempotent) 사용하세요.
- `dialog`: 양식이 [`<dialog>`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/dialog) 안에 있을 때, 양식을 제출하면 대화상자를 닫습니다.

버튼의 양식 소유자에 지정한 [`method`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#attr-method) 특성보다 우선합니다.

</dd>

<Term>formnovalidate</Term>
<dd>

버튼이 (*양식 소유자*가 존재하고, `type="button"`을 지정하지 않은) 제출용 버튼일 때, 제출 시 양식의 데이터를 [검증](https://developer.mozilla.org/en-US/docs/Learn/Forms/Form_validation)하지 않음을 지정하는 불리언 특성입니다. 버튼의 양식 소유자에 지정한 [`novalidate`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#attr-novalidate) 특성보다 우선합니다.

</dd>

<Term>formtarget</Term>
<dd>

버튼이 (*양식 소유자*가 존재하고, `type="button"`을 지정하지 않은) 제출용 버튼일 때, 제출 결과를 표시할 위치를 지정합니다. 가능한 값은 브라우징 맥락(탭, 창, <code>&lt;iframe&gt;</code>)의 이름입니다.

다음의 키워드는 특별한 뜻을 가지고 있습니다.

- `_self` (기본값): 결과를 현재 브라우징 맥락에 보여줍니다.
- `_blank`: 결과를 새롭게 생성한 이름 없는 브라우징 맥락에 보여줍니다.
- `_parent`: 현재 페이지가 프레임 안에 존재하는 경우, 결과를 현재 브라우징 맥락의 부모에 보여줍니다. 부모가 없으면 `_self`와 동일합니다.
- `_top`: 결과를 최상위 브라우징 맥락에 보여줍니다. 최상위 브라우징 맥락이란 현재 맥락의 조상 중 부모가 없는 맥락입니다. 현재 맥락이 부모를 가지지 않으면 `_self`와 동일합니다.

버튼의 양식 소유자에 지정한 [`target`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#attr-target) 특성보다 우선합니다.

</dd>

<Term>name</Term>
<dd>

버튼의 이름입니다. 양식을 제출할 때 버튼의 [`value`](#term-value)와 쌍을 이뤄 데이터에 포함됩니다.

</dd>

<Term>type</Term>
<dd>

버튼의 기본 동작입니다. 가능한 값은 다음과 같습니다.

- `submit`: 버튼이 양식의 데이터를 서버로 제출합니다. `<form>`과 연결된 버튼에 `type`을 지정하지 않았거나 유효하지 않은 값을 지정했을 때의 기본 값입니다.
- `reset`: [`<input type="reset">`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input/reset)처럼, 클릭했을 때 양식의 모든 컨트롤을 기본 값으로 초기화합니다. 초기화 여부를 따로 사용자에게 물어보지 않기 때문에 조심해야 합니다.
- `button`: 버튼이 기본 동작을 가지지 않습니다. 따라서 클릭해도 아무런 동작을 하지 않습니다. 대신, 클라이언트 사이드 스크립트에서 이벤트를 수신해서 원하는 동작을 하도록 설정할 수 있습니다.

</dd>

<Term>value</Term>
<dd>

버튼의 값입니다. 양식을 제출할 때 버튼의 [`name`](#term-name)과 쌍을 이뤄 데이터에 포함됩니다.

</dd>

</dl>

# 참고

[`<input>`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input)과는 달리, `<button>`에는 내부에 HTML 콘텐츠([`<i>`](./i), [`<br>`](./br), [`<img>`](./img)까지)를 넣을 수 있고, [`::after`](https://developer.mozilla.org/en-US/docs/Web/CSS/::after)와 [`::before`](https://developer.mozilla.org/en-US/docs/Web/CSS/::before) 의사 요소를 사용해서 복잡한 렌더링도 추가할 수 있습니다.

버튼을 제출 용도로 추가하는 것이 아니면 `type` 특성을 `button`으로 설정하세요. 그렇지 않으면 의도하지 않은 데이터 제출로 인해 정상적인 응답을 받지 못하고, (존재하지 않을 수도 있는) 그 응답을 불러오려고 시도하는 과정에서 문서의 현재 상태가 사라질 수 있습니다.

# 접근성 고려사항

## 아이콘 버튼

아이콘만 사용하는 버튼에는 *접근 가능한 이름*이 없습니다. 스크린 리더와 같은 보조 기술은 문서를 분석하고 [접근성 트리](https://developer.mozilla.org/en-US/docs/Learn/Accessibility/What_is_accessibility#accessibility_apis)를 생성할 때 접근 가능한 이름에 의존합니다. 보조 기술은 페이지의 콘텐츠를 탐색하고 조작할 때 트리를 사용합니다.

아이콘 버튼에 접근 가능한 이름을 제공하려면 버튼의 기능을 간결하게 설명하는 텍스트를 `<button>` 안에 배치하세요.

```html
<button type="button">
  <img src="/assets/icons/favorite.svg" aria-hidden>
  즐겨찾기에 추가
</button>
```

텍스트가 보이지는 않게 하고 싶을 때, [여러 CSS 속성을 조합](https://gomakethings.com/hidden-content-for-better-a11y/#hiding-the-link)하면 보조 기술은 읽을 수 있지만 시각적으로는 보이지 않는 텍스트를 만들 수 있습니다.

그러나, 아이콘의 의미나 해당 버튼의 용도에 익숙하지 않은 사용자 역시 텍스트를 읽을 수 있다면 도움이 된다는 사실을 기억하는 것이 좋습니다. 특히 기술에 친숙하지 않거나, 문화권이 달라서 아이콘을 다르게 받아들일 수 있는 사용자에게 있어 보조 텍스트의 존재 유무는 큰 차이를 만듭니다.

## 크기와 간격

### 크기

버튼과 같은 대화형 요소는 상호작용하기 충분한 크기의 활성화 영역을 가져야 합니다. 운동 조절 장애를 가진 사용자뿐만 아니라 터치스크린처럼 정확하지 않은 입력 도구 사용자도 충분한 활성화 영역 크기에 도움을 받습니다. 최소 44x44의 CSS 픽셀 크기를 권고합니다.

### 간격

버튼을 포함한 대화형 요소 다수를 시각적으로 가까이 배치할 땐 서로를 분리하는 공간을 둬야 합니다. 운동 조절 장애를 가진 사용자가 원하지 않는 요소와 상호작용하는 것을 방지할 수 있습니다.
