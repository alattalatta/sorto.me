**HTML 전역 특성**<small>(전역 속성)</small>은 모든 HTML 요소가 지원하는 특성으로, 모든 요소에 사용할 수 있으나 일부 요소에서는 아무런 효과도 없을 수 있습니다.

전역 특성은 [모든 HTML 요소](./Element)에 지정할 수 있으며, 심지어 표준에 명시되지 않은 요소에도 지정할 수 있습니다. 비록 비표준 요소를 사용하는 순간 문서는 HTML 표준을 준수하지 않는 것이지만, 그렇다고 해도 모든 비표준 요소는 전역 특성을 허용해야 합니다. 예를 들어, `<foo>`라는 이름의 유효하지 않은 HTML 태그를 사용한 문서에서도, HTML 표준을 준수하는 브라우저라면 <TermLink>hidden</TermLink> 특성을 지정한 `<foo hidden>...</foo>` 콘텐츠를 화면에 그리지 않습니다.

기본 HTML 전역 특성 외에 다음과 같은 전역 특성도 존재합니다.

- `xml:lang`과 `xml:base`는 XHTML 명세에서 상속받은 것으로 이제 사용하지 않지만, 호환성 유지를 위해 아직 존재합니다.
- 접근성 향상을 위해 사용하는 다수의 [`aria-*`](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA) 특성도 있습니다.
- [이벤트 처리기](https://developer.mozilla.org/en-US/docs/Web/Guide/Events/Event_handlers) 특성: `onabort`, `onautocomplete`, `onautocompleteerror`, `onblur`, `oncancel`, `oncanplay`, `oncanplaythrough`, `onchange`, `onclick`, `onclose`, `oncontextmenu`, `oncuechange`, `ondblclick`, `ondrag`, `ondragend`, `ondragenter`, `ondragexit`, `ondragleave`, `ondragover`, `ondragstart`, `ondrop`, `ondurationchange`, `onemptied`, `onended`, `onerror`, `onfocus`, `oninput`, `oninvalid`, `onkeydown`, `onkeypress`, `onkeyup`, `onload`, `onloadeddata`, `onloadedmetadata`, `onloadstart`, `onmousedown`, `onmouseenter`, `onmouseleave`, `onmousemove`, `onmouseout`, `onmouseover`, `onmouseup`, `onmousewheel`, `onpause`, `onplay`, `onplaying`, `onprogress`, `onratechange`, `onreset`, `onresize`, `onscroll`, `onseeked`, `onseeking`, `onselect`, `onshow`, `onsort`, `onstalled`, `onsubmit`, `onsuspend`, `ontimeupdate`, `ontoggle`, `onvolumechange`, `onwaiting`.

# 전역 특성 목록

## 일반 특성

<dl>
  <Term>accesskey</Term>
  <dd>
    현재 요소에 대한 키보드 단축키 생성을 위한 힌트를 제공합니다. 이 특성은 공백으로 구분한 문자 목록으로 구성됩니다.
    브라우저는 주어진 여러 개의 값 중 장치의 키보드 레이아웃에 존재하는 첫 번째 키를 사용해야 합니다.
  </dd>
  <Term href="./Global_attributes/autocapitalize">autocapitalize</Term>
  <dd>
    <p>
      사용자가 입력 또는 수정하는 텍스트를 자동으로 대문자로 변환할지 여부와 변환하는 방법을 제어합니다. 다음 값을
      사용해야 합니다.
    </p>
    <ul>
      <li>
        <code>off</code> 또는 <code>none</code>: 대소문자 자동 변환이 발생하지 않습니다. (모든 글자의 기본값이 소문자)
      </li>
      <li>
        <code>on</code> 또는 <code>sentences</code>: 각 문장의 첫 문자를 대문자로 변환합니다. 나머지 글자의 기본값은
        소문자입니다.
      </li>
      <li>
        <code>words</code>: 각 단어의 첫 글자를 대문자로 변환합니다. 나머지 글자의 기본값은 소문자입니다.
      </li>
      <li>
        <code>characters</code>: 모든 글자의 기본값이 대문자입니다.
      </li>
    </ul>
  </dd>
  <Term href="./Global_attributes/class">class</Term>
  <dd>
    요소의 클래스를 나타내는 공백 구분 목록입니다. 요소 클래스를 사용하면 CSS{' '}
    <Anchor href="https://developer.mozilla.org/en-US/docs/Web/CSS/Class_selectors">클래스 선택자</Anchor>와 JavaScript의
    <Anchor href="https://developer.mozilla.org/en-US/docs/Web/API/Document/getElementsByClassName">
      <code>Document.getElementsByClassName()</code>
    </Anchor> 등의 함수로 특정한 요소를 구별해 접근할 수 있습니다.
  </dd>
  <Term>contenteditable</Term>
  <dd>
    <p>
      해당 요소를 사용자가 편집할 수 있는지 지정합니다. 편집 가능한 경우, 브라우저는 해당 요소의 위젯을 수정해 편집
      가능한 상태로 만듭니다. 다음 값을 사용해야 합니다.
    </p>
    <ul>
      <li>
        <code>true</code> 또는 빈 문자열: 요소가 편집 가능합니다.
      </li>
      <li>
        <code>false</code>: 요소를 편집할 수 없습니다.
      </li>
    </ul>
  </dd>
  <Term>data-*</Term>
  <dd>
    사용자 지정 데이터 특성(custom data attribute)이라는 특성 클래스를 형성해서, 임의의 데이터를 HTML과 DOM 사이에서
    교환할 수 있도록 합니다. 모든 커스텀 데이터는 해당 요소의{' '}
    <Anchor href="https://developer.mozilla.org/en-US/docs/Web/API/HTMLElement">
      <code>HTMLElement</code>
    </Anchor>{' '}
    인터페이스 내{' '}
    <Anchor href="https://developer.mozilla.org/en-US/docs/Web/API/HTMLOrForeignElement/dataset">
      <code>dataset</code>
    </Anchor>{' '}
    속성을 통해 접근할 수 있습니다.
  </dd>
  <Term>dir</Term>
  <dd>
    <p>요소의 쓰기 방향을 지정합니다. 다음 값을 사용해야 합니다.</p>
    <ul>
      <li>
        <code>ltr</code>: 왼쪽에서 시작해 오른쪽으로 씁니다. 한국어와 영어가 왼쪽에서 오른쪽으로 쓰는 언어입니다.
      </li>
      <li>
        <code>rtl</code>: 오른쪽에서 시작해 왼쪽으로 씁니다. 아랍어가 오른쪽에서 왼쪽으로 쓰는 언어입니다.
      </li>
      <li>
        <code>auto</code>: 사용자 에이전트가 쓰기 방향을 결정합니다. 요소 내의 문자를 순회하다가, 뚜렷한 쓰기 방향을
        가진 문자를 마주치면 해당 방향을 요소에 적용하는 기본적인 알고리즘을 사용합니다.
      </li>
    </ul>
  </dd>
  <Term>draggable</Term>
  <dd>
    <p>
      <Anchor href="https://developer.mozilla.org/en-US/docs/Web/API/HTML_Drag_and_Drop_API">Drag and Drop API</Anchor>를
      사용해 요소를 드래그할 수 있는지 지정합니다. 다음 값을 사용해야 합니다.
    </p>
    <ul>
      <li>
        <code>true</code>: 요소를 드래그할 수 있습니다.
      </li>
      <li>
        <code>false</code>: 요소를 드래그할 수 없습니다.
      </li>
    </ul>
  </dd>
  <Term>enterkeyhint</Term>
  <dd>가상 키보드의 엔터 키에 표시할 액션 레이블 또는 아이콘을 지정합니다.</dd>
  <Term>hidden</Term>
  <dd>
    해당 요소가 현재 문서에서 아직 관련이 없거나, 더 이상 관련이 없어졌음을 나타내는 불리언 특성입니다.
    예를 들어 로그인 후에 보여줄 부분을 로그인 전인 사용자에게서 숨길 때 사용할 수 있습니다.
    브라우저는 <code>hidden</code> 특성을 가진 요소를 렌더링 하지 않습니다. 현재 문서에 관련 있는 요소를
    오직 화면에서 숨기기 위한 용도로 사용해서는 안됩니다.
  </dd>
  <Term>id</Term>
  <dd>
    문서 내에서 유일해야 하는 고유 식별자(ID)를 지정합니다. 프래그먼트 식별자(<code>#</code>)를 사용한 링크, 스크립트,
    CSS 스타일링 등 요소를 식별할 때 사용합니다.
  </dd>
  <Term>inputmode</Term>
  <dd>
    사용자가 요소나 요소의 콘텐츠를 편집할 때 제공해야 할 가상 키보드 설정을 브라우저에 알려줍니다. 주로{' '}
    <code>&lt;input&gt;</code> 요소에 사용하지만, <TermLink>contenteditable</TermLink> 특성을 가진 요소라면 종류
    불문하고 사용할 수 있습니다.
  </dd>
  <Term>is</Term>
  <dd>
    표준 HTML 요소가 사용자 정의 요소처럼 동작하도록 지정합니다. (자세한 내용은{' '}
    <Anchor href="https://developer.mozilla.org/en-US/docs/Web/Web_Components/Using_custom_elements">
      사용자 정의 요소 사용하기
    </Anchor>
    를 참고하세요)
  </dd>
  <Term>lang</Term>
  <dd>
    편집할 수 없는 요소의 언어, 또는 편집할 수 있는 요소에서 사용자가 사용해야 하는 언어를 알려줍니다.{' '}
    <Anchor href="https://www.ietf.org/rfc/bcp/bcp47.txt">BCP47</Anchor>이 정의하는 언어 식별 태그 하나를 값으로
    사용해야 합니다.
  </dd>
  <Term>nonce</Term>
  <dd>
    논스(일회용 숫자 값)을 지정합니다. 주어진 값은{' '}
    <Anchor href="https://developer.mozilla.org/en-US/docs/Web/HTTP/CSP">콘텐츠 보안 정책</Anchor>에서 사용해서 주어진
    리소스를 허용할지 결정합니다.
  </dd>
  <Term>part</Term>
  <dd>
    요소의 파트 이름입니다. 공백으로 구분한 리스트입니다. 파트 이름을 사용하면 CSS의 <code>::part</code> 의사 요소를
    사용해서 섀도 트리 내의 특정 요소만 선택해 스타일을 적용할 수 있습니다.
  </dd>
  <Term>slot</Term>
  <dd>
    <Anchor href="https://developer.mozilla.org/en-US/docs/Web/Web_Components/Using_shadow_DOM">섀도 DOM</Anchor> 섀도
    트리의 슬롯을 요소에 할당합니다. <code>&lt;slot&gt;</code> 중 <code>name</code> 특성의 값이 <code>slot</code>에
    지정한 값과 일치하는 슬롯에 배치됩니다.
  </dd>
  <Term>spellcheck</Term>
  <dd>
    <p>요소의 맞춤법 검사 여부를 지정합니다. 다음 값을 사용해야 합니다.</p>
    <ul>
      <li>
        <code>true</code>: 가능하다면 맞춤법 검사를 수행해야 함을 나타냅니다.
      </li>
      <li>
        <code>false</code>: 맞춤법 검사를 수행하지 않아야 함을 나타냅니다.
      </li>
    </ul>
  </dd>
  <Term>style</Term>
  <dd>요소에 적용할 CSS 스타일 선언을 지정합니다.</dd>
  <Term>tabindex</Term>
  <dd>
    <p>
      요소가 입력 포커스를 받을 수 있는지, 순차적 키보드 탐색에 참여하는지, 참여한다면 어떤 순서로 참여하는지 지정하는
      정수형 특성입니다. 다음 값을 사용해야 합니다.
    </p>
    <ul>
      <li>음의 정수: 요소가 포커스를 받을 수 있지만 순차적 키보드 탐색으로는 접근할 수 없음을 나타냅니다.</li>
      <li>
        <code>0</code>: 요소가 포커스를 받을 수 있으며 순차적 키보드 탐색으로 접근할 수 있지만, 그 순서는 플랫폼 관례에
        따름을 나타냅니다.
      </li>
      <li>
        양의 정수: 요소가 포커스를 받을 수 있고 순차적 키보드 탐색으로 접근할 수 있습니다. 탐색 순서는{' '}
        <code>tabindex</code> 값의 오름차순입니다. 여러 요소가 같은 <code>tabindex</code> 값을 공유하면 그 순서는 문서
        내의 상대적 위치와 동일합니다.
      </li>
    </ul>
  </dd>
  <Term>title</Term>
  <dd>요소에 대한 추가 정보를 제공하는 텍스트입니다. 사용자에게는 보통 툴팁으로 보여집니다.</dd>
  <Term>translate</Term>
  <dd>
    <p>
      페이지를 번역할 때, 요소의{' '}
      <Anchor href="https://developer.mozilla.org/en-US/docs/Web/API/Text">
        <code>Text</code>
      </Anchor>{' '}
      노드 자식 및 번역 가능한 특성의 값을 번역해야 하는지, 아니면 그대로 둬야 하는지 지정합니다. 다음 값을 사용해야
      합니다.
    </p>
    <ul>
      <li>
        <code>yes</code> 또는 빈 문자열: 요소를 번역합니다.
      </li>
      <li>
        <code>no</code>: 요소를 번역하지 않습니다.
      </li>
    </ul>
  </dd>
</dl>

<dl>
  <Term>itemid</Term>
  <dd>아이템의 고유하고 전역적인 식별자입니다.</dd>
  <Term>itemprop</Term>
  <dd>
    아이템에 속성을 추가합니다. 모든 HTML 요소에 <code>itemprop</code> 요소를 지정할 수 있습니다.
  </dd>
  <Term>itemref</Term>
  <dd>
    <code>itemscope</code> 특성을 가진 요소를 해당 요소의 자손이 아닌 요소와 연결할 때 사용합니다. 요소의{' '}
    <TermLink>id</TermLink>(<code>itemid</code>가 아님)를 공백 구분 리스트로 지정해야 합니다.
  </dd>
  <Term>itemscope</Term>
  <dd>
    (대개) <code>itemtype</code>과 함께 사용해서, 이 요소에 속한 HTML이 특정 아이템을 서술함을 나타냅니다.
  </dd>
  <Term>itemtype</Term>
  <dd>
    <code>itemprop</code>을 정의할 어휘를 가리키는 URL을 지정합니다. 어휘의 적용 범위는 <code>itemscope</code>가
    정의합니다.
  </dd>
</dl>
