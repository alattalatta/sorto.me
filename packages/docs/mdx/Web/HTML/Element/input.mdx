**HTML `<input>` 요소**는 웹 기반 양식에서 사용자에게서 입력을 받기 위한 상호작용 컨트롤을 생성합니다. 사용자 에이전트에 따라서 다양한 종류의 입력 데이터와 위젯을 사용할 수 있습니다. `<input>`은 입력 유형과 다양한 특성의 조합으로 인해 HTML에서 가장 강력하면서도 가장 복잡한 요소입니다.

<HTMLDemo height={120}>

```html
<label for="name">이름 (2 ~ 8글자):</label>

<input type="text" id="name" name="name" required minlength="2" maxlength="8" size="10" />
```

```css
label {
  display: block;
  margin-bottom: 8px;
}
```

</HTMLDemo>

# `<input>` 유형

`<input>`의 동작 방식은 많은 부분을 **`type`** 특성의 값이 결정합니다. `type`을 따로 지정하지 않았을 때의 기본값은 `text`입니다.

사용 가능한 값은 다음과 같습니다.

| 유형             | 설명                                                                                                                                                     | 미리보기                           |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------- |
| `button`         | `value` 특성을 보여주는 것 외에는 기본 동작이 없는, 단순한 버튼입니다.                                                                                   | <input type="button" />            |
| `checkbox`       | 하나의 값을 선택하거나 선택하지 않을 수 있는 체크박스입니다.                                                                                             | <input type="checkbox" />          |
| `color`          | 색을 지정하는 입력 칸입니다. 지원하는 브라우저에서는 색상 선택기를 보여줍니다.                                                                           | <input type="color" />             |
| `date`           | 시간 없는 날짜(연/월/일)를 입력하는 칸입니다. 지원하는 브라우저에서는 달력이나 날짜 휠을 보여줍니다.                                                     | <input type="date" />              |
| `datetime-local` | 시간을 포함한 날짜를 입력하는 칸입니다. 시간대는 입력할 수 없습니다. 지원하는 브라우저에서는 달력과 시계 등을 보여줍니다.                                | <input type="datetime-local" />    |
| `email`          | 이메일을 입력하는 칸입니다. 텍스트 입력 칸과 동일한 모습이지만, 지원하는 브라우저에서 값 검증을, 동적 키보드 환경에서 관련 키보드를 보여줍니다.          | <input type="email" />             |
| `file`           | 사용자가 파일을 선택할 수 있는 칸입니다. [`accept`](#term-accept) 특성으로 선택 가능한 파일 유형을 지정할 수 있습니다.                                   | <input type="file" />              |
| `hidden`         | 사용자가 볼 수는 없지만, 값은 서버로 제출하는 숨겨진 칸입니다. 오른쪽에 미리보기가 있지만 보실 수 없겠네요!                                              | <input type="hidden" />            |
| `image`          | 이미지를 보여주는 제출 버튼입니다. `src`로 보여줄 이미지를, `alt`로 대체 텍스트를 지정합니다.                                                            | <input type="image" alt='제출!' /> |
| `month`          | 연/월을 입력하는 칸입니다.                                                                                                                               | <input type="month" />             |
| `number`         | 숫자를 입력하는 칸입니다. 값을 조절할 수 있는 위아래 화살표, 그리고 지원할 경우 값 검증을 포함합니다. 동적 키보드 환경에서 숫자 키패드를 보여줍니다.     | <input type="number" />            |
| `password`       | 값을 볼 수 없는 텍스트 입력 칸입니다. 웹 사이트가 안전하지 않을 경우 경고를 표시합니다.                                                                  | <input type="password" />          |
| `radio`          | 같은 [`name`](#term-name)을 가진 다수의 선택지 중 하나를 선택할 수 있는 라디오 버튼입니다.                                                               | <input type="radio" />             |
| `range`          | 정확하지 않아도 되는 숫자 입력 칸입니다. [`min`](#term-min)과 [`max`](#term-max) 특성으로 최소와 최대 값을 설정할 수 있습니다.                           | <input type="range" />             |
| `reset`          | 양식의 모든 값을 기본으로 초기화하는 버튼입니다.                                                                                                         | <input type="reset" />             |
| `search`         | 검색 문자열을 입력하는 칸입니다. 지원하는 브라우저에서는 초기화 버튼을 포함할 수 있습니다. 동적 키보드 환경에서는 엔터 키 대신 검색 아이콘을 보여줍니다. | <input type="search" />            |
| `submit`         | 양식을 제출하는 버튼입니다.                                                                                                                              | <input type="submit" />            |
| `tel`            | 전화번호를 입력하는 칸입니다. 동적 키보드 환경에서는 전화번호 키패드를 보여줍니다.                                                                       | <input type="tel" />               |
| `text`           | 한 줄의 텍스트를 입력하는 칸입니다.                                                                                                                      | <input type="text" />              |
| `time`           | 시간을 입력하는 칸입니다. 시간대는 입력할 수 없습니다.                                                                                                   | <input type="time" />              |
| `url`            | URL을 입력하는 칸입니다. 텍스트 입력 칸과 동일한 모습이지만, 지원하는 브라우저에서 값 검증을, 동적 키보드 환경에서 관련 키보드를 보여줍니다.             | <input type="url" />               |
| `week`           | 연도의 주차를 입력하는 칸입니다. 시간대는 입력할 수 없습니다.                                                                                            | <input type="week" />              |

# 특성

`<input>`의 강력함은 많은 부분이 특성에서 나오고, 바로 위에서 설명한 `type` 특성이 그 중에서도 제일 중요합니다. 모든 `<input>`은 유형에 관계 없이 [`HTMLInputElement`](https://developer.mozilla.org/en-US/docs/Web/API/HTMLInputElement)를 기반으로 하므로, 기술적으로는 모든 유형이 동일한 특성을 공유합니다. 그러나 실제로는, 대부분의 특성은 일부 `type`에서면 효과가 있습니다. 이에 더해 어떤 특성은 `type`에 따라 그 효과가 달라지기도 합니다.

아래의 표는 가능한 모든 특성을 간단한 설명과, 효과가 있는 유형을 나열합니다. 특성별 자세한 설명은 더 아래에서 확인할 수 있습니다.

`<input>`은 [전역 특성](/docs/Web/HTML/Global_attributes)을 포함합니다.

| 특성                                     | 유형                                       | 설명                                                          |
| ---------------------------------------- | ------------------------------------------ | ------------------------------------------------------------- |
| [`accept`](#term-accept)                 | `file`                                     | 업로드 컨트롤에서 선택 가능한 파일 유형                       |
| [`alt`](#term-alt)                       | `image`                                    | 이미지를 보여줄 수 없을 때의 대체 텍스트                      |
| [`autocomplete`](#term-autocomplete)     | 모두                                       | 양식 자동완성 기능 힌트                                       |
| [`autofocus`](#term-autofocus)           | 모두                                       | 페이지를 불러온 후 지정한 양식에 자동으로 포커스              |
| [`capture`](#term-capture)               | `file`                                     | 업로드 컨트롤에 미디어 캡처 선택 가능                         |
| [`checked`](#term-checked)               | `checkbox`, `radio`                        | 컨트롤의 체크 여부                                            |
| [`dirname`](#term-dirname)               | `search`, `text`                           | 양식 제출 시 값의 방향성 정보 필드의 이름                     |
| [`disabled`](#term-disabled)             | 모두                                       | 컨트롤의 비활성 여부                                          |
| [`form`](#term-form)                     | 모두                                       | 컨트롤을 양식 요소와 연결                                     |
| [`formaction`](#term-formaction)         | `image`, `submit`                          | 양식 제출 시 데이터를 전송할 URL                              |
| [`formenctype`](#term-formenctype)       | `image`, `submit`                          | 양식 제출 시 사용할 데이터 인코딩 유형                        |
| [`formmethod`](#term-formmethod)         | `image`, `submit`                          | 양식 제출 시 사용할 HTTP 메서드                               |
| [`formnovalidate`](#term-formnovalidate) | `image`, `submit`                          | 양식 제출 시 값 검증 안함                                     |
| [`formtarget`](#term-formaction)         | `image`, `submit`                          | 양식 제출 결과를 보여줄 브라우징 맥락                         |
| [`height`](#term-height)                 | `image`                                    | 이미지 높이                                                   |
| [`list`](#term-list)                     | 대부분                                     | 자동완성 선택지로 사용할 [`<datalist>`](./datalist) 요소의 ID |
| [`max`](#term-max)                       | 숫자 입력                                  | 최대 값                                                       |
| [`maxlength`](#term-maxlength)           | `password`, `search`, `tel`, `text`, `url` | 값의 최대 길이                                                |
| [`min`](#term-min)                       | 숫자 입력                                  | 최소 값                                                       |
| [`minlength`](#term-minlength)           | `password`, `search`, `tel`, `text`, `url` | 값의 최소 길이                                                |
| [`multiple`](#term-multiple)             | `email`, `file`                            | 다수의 값 허용 여부                                           |
| [`name`](#term-name)                     | 모두                                       | 컨트롤의 이름, 양식 제출 시 이름/값 쌍의 이름으로 제출        |
| [`pattern`](#term-pattern)               | `password`, `tel`, `text`                  | 값이 만족해야 하는 패턴 (정규표현식)                          |
| [`placeholder`](#term-placeholder)       | `password`, `search`, `tel`, `text`, `url` | 입력한 값이 아직 없을 때 보여줄 텍스트                        |
| [`readonly`](#term-readonly)             | 대부분                                     | 값을 편집할 수 없음                                           |
| [`required`](#term-required)             | 대부분                                     | 값을 반드시 지정해야 제출할 수 있음                           |
| [`size`](#term-size)                     | `email`, `password`, `tel`, `text`         | 컨트롤의 크기                                                 |
| [`type`](#term-type)                     | 모두                                       | 컨트롤의 유형                                                 |
| [`value`](#term-value)                   | 모두                                       | 컨트롤의 초기 값                                              |
| [`width`](#term-width)                   | `image`                                    | 이미지 너비                                                   |

## 개별 특성

<dl>
<Term>accept</Term>
<dd>

`file` 유형에만 사용할 수 있습니다. 업로드 컨트롤에서 사용자가 선택할 수 있는 파일 유형을 지정합니다.

</dd>

<Term>alt</Term>
<dd>

`image` 유형에만 사용할 수 있습니다. `image`의 [`src`](#term-src)가 없거나, 불러오지 못했을 때 대신 보여줄 대체 텍스트입니다.

</dd>

<Term>autocomplete</Term>
<dd>

입력 칸이 제공해야 할 자동완성 유형을 지정하는 공백 구분 목록입니다. 일반적인 자동완성은 사용자가 동일한 입력 칸에 넣었던 값을 기억했다가 이후에 제공하는 것이지만, 사용자 장치의 연락처 목록을 `email` 자동완성으로 제공하는 등 사용자 에이전트에 따라 더 복잡한 방식의 자동완성도 존재할 수 있습니다. [HTML 자동완성 특성](https://developer.mozilla.org/en-US/docs/Web/HTML/Attributes/autocomplete)에서 더 자세히 알아보세요.

`autocomplete` 특성은 숫자나 문자 값을 사용하는 `hidden`, `text`, `search`, `url`, `tel`, `email`, `date`, `month`, `week`, `time`, `datetime-local`, `number`, `range`, `color`, `password`에 유효하고, `checkbox`, `radio`, `file`, 버튼 유형에는 유효하지 않습니다.

</dd>

<Term>autofocus</Term>
<dd>

페이지 불러오기가 끝나는 순간, 또는 이 입력 칸을 포함한 [`<dialog>`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/dialog) 요소가 보여지는 순간 이 입력 칸으로 포커스를 이동해야 하는지 지정하는 불리언 특성입니다.

<Callout.Root childAs="div" color="warn" label="참고">

`autofocus` 특성을 포함한 요소는 [`DOMContentLoaded`](https://developer.mozilla.org/en-US/docs/Web/API/Window/DOMContentLoaded_event) 이벤트가 발생하기 전에 포커스를 획득할 수도 있습니다.

</Callout.Root>

문서 내에서 단 하나의 요소만 `autofocus` 특성을 가질 수 있습니다. 두 개 이상의 요소에 지정한 경우 더 앞에 위치한 요소가 포커스를 획득합니다.

`hidden` 유형에는 포커스를 할 수 없으므로 `autofocus`도 유효하지 않습니다.

<Callout.Root childAs="div" color="warn" label="주의">

`autofocus`를 사용한 자동 포커스 이동에는 사전 알림 등이 없으므로 스크린 리더 사용자나 인지력 저하를 겪는 사용자를 방해할 수 있습니다.

</Callout.Root>

`autofocus` 특성을 적용하기 전에 접근성에 대해 깊게 고려해보세요. 자동으로 포커스를 이동하는 과정에 페이지 스크롤이 발생할 수 있으며, 일부 터치 장치에서는 원하지 않아도 즉시 가상 키보드가 노출될 수 있습니다. 스크린 리더는 포커스를 받은 컨트롤의 레이블은 발화하지만 그 전의 내용은 아무것도 말해주지 않고, 시각에 문제가 없는 사용자라도 작은 화면을 사용하고 있는 경우 이전 내용을 동일하게 놓칠 것입니다.

</dd>

<Term>capture</Term>
<dd>

`file` 유형에만 사용할 수 있습니다. 지정한 미디어(마이크, 카메라 등)에서 새로운 파일을 캡처해 업로드할 수 있습니다.

</dd>

<Term>checked</Term>
<dd>

`checkbox`와 `radio` 유형에 사용할 수 있습니다. 불리언 특성으로, `radio`에 지정한 경우 같은 이름으로 이루어진 그룹 내에서 이 라디오 버튼을 선택했음을 나타냅니다. `checkbox`에 지정한 경우 기본적(페이지 로드 시)으로 이 체크박스를 체크한 상태로 지정합니다. `checked` 특성은 _현재_ 체크 상태를 나타내지 않습니다. 체크박스와 라디오 버튼의 상태가 바뀌어도 `checked` 콘텐츠 특성은 변화를 반영하지 않습니다. (현재 체크 상태는 [`HTMLInputElement`](https://developer.mozilla.org/en-US/docs/Web/API/HTMLInputElement)의 `checked` IDL 특성이 나타냅니다.)

<Callout.Root childAs="div" color="warn" label="참고">

다른 입력 칸과 달리, 체크박스와 라디오의 값은 `checked`일 때만 서버로 제출됩니다.

예를 들어, `name`이 `fruit`이고 `value`가 `cherry`인 체크박스를 체크한 상황이면 양식 데이터에 `fruit=cherry`를 포함합니다. 그러나 체크박스를 체크하지 않은 상황일 땐 `fruit`이 양식 데이터에 아예 포함되지 않습니다. 체크박스와 라디오의 기본 `value`는 `on`입니다.

</Callout.Root>

</dd>

<Term>dirname</Term>
<dd>

`search`와 `text` 유형에 사용할 수 있습니다. 지정하면 요소의 쓰기 방향을 제출 데이터에 포함합니다. 따라서 제출 데이터에는 두 개의 이름/값 쌍이 포함되는데, 첫 번째는 [`name`](#term-name)/[`value`](#term-value) 쌍이고, 두 번째는 `dirname`/(쓰기 방향에 따라)`ltr` 또는 `rtl`입니다.

```html
<form action="page.html" method="post">
  <label>Fruit: <input type="text" name="fruit" dirname="fruit.dir" value="cherry" /></label>
  <button type="submit">제출</button>
</form>
```

위의 예제 양식을 제출하면 데이터의 `fruit`은 `cherry`를, `fruit.dir`는 (한국어의 기본 쓰기 방향인) `ltr`을 가지게 됩니다.

</dd>

<Term>disabled</Term>
<dd>

지정할 경우 사용자와의 상호작용을 막는 불리언 특성입니다. 비활성 컨트롤은 사용 불가능하다는 것을 나타내기 위해 더 연한 색상을 사용하는 등 모양이 달라지곤 합니다.

상호작용을 막는다는 것은 컨트롤이 클릭 이벤트를 받지 못하고, 양식 제출 시 컨트롤의 값을 포함하지 않는다는 뜻입니다.

</dd>

<Term>form</Term>
<dd>

이 컨트롤을 연결할 _양식 소유자_([`<form>`](./form) 요소)의 ID를 지정합니다. 같은 문서 상에 존재하는 `<form>`의 `id` 특성을 지정할 수 있습니다. 이 특성이 존재하지 않으면 `<input>`의 양식 소유자는 가장 가까운 조상 `<form>`이 됩니다. (존재할 경우)

`form` 특성을 사용하면 `<input>`이 `<form>`의 자손이 아니어도 연결할 수 있습니다. 조상에 `<form>`이 있는데 `form` 특성을 지정한 경우 특성이 가리키는 `<form>`이 우선합니다.

</dd>

<Term>formaction</Term>
<dd>

`image`와 `submit` 유형에 사용할 수 있습니다. 버튼을 클릭했을 때 데이터를 제출할 URL을 지정합니다. 버튼의 *양식 소유자*에 지정한 [`action`](./form#term-action) 특성보다 우선합니다. 양식 소유자가 없으면 아무 효과도 없습니다.

</dd>

<Term>formenctype</Term>
<dd>

`image`와 `submit` 유형에 사용할 수 있습니다. 버튼을 클릭했을 때, 제출할 데이터를 인코딩하는 방법을 지정합니다. 가능한 값은 다음과 같습니다.

- `application/x-www-form-urlencoded`: 기본값입니다.
- `multipart/form-data`: [`type`](#term-type)이 `file`인 `<input>`을 제출할 때 사용합니다.
- `text/plain`: 디버깅 용도로 사용하는 값입니다. 실제 사용자가 제출하는 데이터의 인코딩에는 사용하지 말아야 합니다.

버튼의 양식 소유자에 지정한 [`enctype`](./form#term-enctype) 특성보다 우선합니다.

</dd>

<Term>formmethod</Term>
<dd>

`image`와 `submit` 유형에 사용할 수 있습니다. 버튼을 클릭했을 때, 제출할 데이터를 전송할 [HTTP 메서드](https://developer.mozilla.org/en-US/docs/Web/HTTP/Methods)를 지정합니다. 가능한 값은 다음과 같습니다.

- `post`: 양식의 데이터를 HTTP 요청의 본문에 넣어서 서버로 전송합니다. 로그인 키처럼 공개해선 안되는 정보를 포함해야 할 때 사용하세요.
- `get`: 양식의 데이터를 `action` URL 뒤의 쿼리 문자열로 붙여서 서버로 전송합니다. 검색 창처럼 [양식이 유발하는 부수효과가 없을 때](https://developer.mozilla.org/ko/docs/Glossary/Idempotent) 사용하세요.
- `dialog`: 양식이 [`<dialog>`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/dialog) 안에 있을 때, 양식을 제출하면 대화상자를 닫습니다.

버튼의 양식 소유자에 지정한 [`method`](./form#term-method) 특성보다 우선합니다.

</dd>

<Term>formnovalidate</Term>
<dd>

`image`와 `submit` 유형에 사용할 수 있습니다. 버튼을 클릭했을 때, 제출 시 양식의 데이터를 [검증](https://developer.mozilla.org/en-US/docs/Learn/Forms/Form_validation)하지 않음을 지정하는 불리언 특성입니다. 버튼의 양식 소유자에 지정한 [`novalidate`](./form#term-novalidate) 특성보다 우선합니다.

</dd>

<Term>formtarget</Term>
<dd>

`image`와 `submit` 유형에 사용할 수 있습니다. 버튼을 클릭했을 때, 제출 결과를 표시할 위치를 지정합니다. 가능한 값은 브라우징 맥락(탭, 창, <code>&lt;iframe&gt;</code>)의 이름입니다.

다음의 키워드는 특별한 뜻을 가지고 있습니다.

- `_self` (기본값): 결과를 현재 브라우징 맥락에 보여줍니다.
- `_blank`: 결과를 새롭게 생성한 이름 없는 브라우징 맥락에 보여줍니다.
- `_parent`: 현재 페이지가 프레임 안에 존재하는 경우, 결과를 현재 브라우징 맥락의 부모에 보여줍니다. 부모가 없으면 `_self`와 동일합니다.
- `_top`: 결과를 최상위 브라우징 맥락에 보여줍니다. 최상위 브라우징 맥락이란 현재 맥락의 조상 중 부모가 없는 맥락입니다. 현재 맥락이 부모를 가지지 않으면 `_self`와 동일합니다.

버튼의 양식 소유자에 지정한 [`target`](./form#term-target) 특성보다 우선합니다.

</dd>

<Term>height</Term>
<dd>

`image` 유형에만 사용할 수 있습니다. 이미지의 높이를 지정합니다.

</dd>

<Term>list</Term>
<dd>

추천 선택지 목록을 나타내는 [`<datalist>`](./datalist) 요소의 ID를 지정합니다. `<input>`의 유형에 사용할 수 없는 선택지는 무시합니다. 추천 선택지는 어디까지나 추천 사항으로, 사용자는 자신이 원하는 다른 값을 입력할 수도 있습니다.

`text`, `search`, `url`, `tel`, `email`, `date`, `month`, `week`, `time`, `datetime-local`, `number`, `range`, `color` 유형에 사용할 수 있습니다.

[`<datalist>` 요소의 예제](./datalist#예제)도 확인해보세요.

</dd>

<Term>max</Term>
<dd>

유효한 최대 값을 지정합니다. 초과하는 값을 받은 경우 이 `<input>`은 [제약 검증](https://developer.mozilla.org/en-US/docs/Web/Guide/HTML/Constraint_validation)을 통과하지 못합니다. `max` 특성에 숫자 외의 값을 지정하면 최대 값을 두지 않습니다.

`date`, `month`, `week`, `time`, `datetime-local`, `number`, `range` 유형에 사용할 수 있습니다.

</dd>

<Term>maxlength</Term>
<dd>

값의 유효한 최대 문자(UTF-16 코드 유닛) 길이를 지정합니다. 0 이상의 정수만 지정할 수 있습니다. 유효하지 않은 값을 지정한 경우 무시합니다.

입력한 텍스트의 길이가 이 값보다 길면 `<input>`이 [제약 검증](https://developer.mozilla.org/en-US/docs/Web/Guide/HTML/Constraint_validation)을 통과하지 못합니다. 기본 상태에서 브라우저는 사용자가 이 특성의 값보다 길게 입력하지 못하도록 막습니다.

`text`, `search`, `url`, `tel`, `email`, `password` 유형에 사용할 수 있습니다.

</dd>

<Term>min</Term>
<dd>

유효한 최소 값을 지정합니다. 미달하는 값을 받은 경우 이 `<input>`은 [제약 검증](https://developer.mozilla.org/en-US/docs/Web/Guide/HTML/Constraint_validation)을 통과하지 못합니다. `min` 특성에 숫자 외의 값을 지정하면 최소 값을 두지 않습니다.

`date`, `month`, `week`, `time`, `datetime-local`, `number`, `range` 유형에 사용할 수 있습니다.

</dd>

<Term>minlength</Term>
<dd>

값의 유효한 최소 문자(UTF-16 코드 유닛) 길이를 지정합니다. 0 이상의 정수만 지정할 수 있습니다. 유효하지 않은 값을 지정한 경우 무시합니다.

입력한 텍스트의 길이가 이 값보다 짧으면 `<input>`이 [제약 검증](https://developer.mozilla.org/en-US/docs/Web/Guide/HTML/Constraint_validation)을 통과하지 못합니다.

`text`, `search`, `url`, `tel`, `email`, `password` 유형에 사용할 수 있습니다.

</dd>

<Term>multiple</Term>
<dd>

`email`과 `file` 유형에 사용할 수 있습니다. 이메일의 경우 다수의 주소를 쉼표로 구분해서, 파일의 경우 여러 파일을 한꺼번에 선택해서 등록할 수 있도록 지정하는 불리언 특성입니다.

</dd>

<Term>name</Term>
<dd>

컨트롤의 이름을 지정합니다. 양식을 제출할 때 값과 함께 이름/값 쌍을 구성합니다. 비록 명세 상으로는 `name`이 필수 특성은 아니지만, 지정하지 않으면 양식 제출 데이터에 포함되지 않으므로 필수로 간주해도 무리가 없습니다.

`name`에는 두 가지 특별한 값을 제외하면 아무 문자열이나 지정할 수 있습니다.

1. `__charset__`: `type="hidden"`인 `<input>`에 지정하면, 사용자 에이전트가 자동으로 `value`를 양식 제출 시 사용한 문자 인코딩으로 설정합니다.
2. `isindex`: 역사적인 이유로 사용할 수 없습니다.

</dd>

<Term>pattern</Term>
<dd>

[제약 검증](https://developer.mozilla.org/en-US/docs/Web/Guide/HTML/Constraint_validation)을 만족하려면 입력 값이 통과해야 하는 정규표현식을 지정합니다. JavaScript [`RegExp`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/RegExp)가 인식할 수 있는 정규표현식이어야 하며, 양 끝의 슬래시는 지정하지 않아야 합니다. 정규표현식을 컴파일할 때 `u` 플래그를 지정하므로 패턴을 ASCII가 아닌 Unicode 코드 포인트로 취급합니다.

유효하지 않은 값을 지정한 경우 무시합니다.

<Callout.Root childAs="div" color="warn" label="참고">

`pattern` 특성을 사용할 땐 어떤 조건에 맞춰 입력해야 하는지 부가 설명 텍스트 등을 사용해 알려주세요.

</Callout.Root>

</dd>

<Term>placeholder</Term>
<dd>

사용자가 입력해야 하는 값을 간략하게 알려줄 텍스트를 지정합니다. 특성 값에 줄바꿈 문자를 포함해서는 안됩니다. `placeholder`는 이 입력 칸의 설명을 제공하는 것이 아니라, 기대하는 데이터의 유형을 나타내야 합니다. 예컨대 레이블이 "이름"인 텍스트 입력 칸의 `placeholder`에 어울리는 값은 "김철수"입니다.

<Callout.Root childAs="div" color="warn" label="참고">

`placeholder`는 양식을 설명할 수 있는 다른 방법에 비해 적합하지 않으며 예상하지 못한 기술적 문제를 유발할 수 있습니다. 아래의 [레이블](#레이블)을 참고하세요.

</Callout.Root>

</dd>

<Term>readonly</Term>
<dd>

사용자가 값을 편집할 수 없도록 하는 불리언 특성입니다.

`text`, `search`, `url`, `tel`, `email`, `date`, `month`, `week`, `time`, `datetime-local`, `number`, `password` 유형에 사용할 수 있습니다.

</dd>

<Term>required</Term>
<dd>

양식을 제출하려면 사용자가 이 입력 칸을 반드시 지정해야 함을 나타내는 불리언 특성입니다.

`text`, `search`, `url`, `tel`, `email`, `date`, `month`, `week`, `time`, `datetime-local`, `number`, `password`, `checkbox`, `radio`, `file` 유형에 사용할 수 있습니다.

</dd>

<Term>size</Term>
<dd>

`email`, `password`, `tel`, `text` 유형에 사용할 수 있습니다. 입력 칸을 얼마나 크게 보여줘야 하는지 지정합니다. 몇 가지 특이점을 제외하면 CSS [`width`](https://developer.mozilla.org/en-US/docs/Web/CSS/width)와 동일한 효과를 냅니다. `size`로 인한 실제 너비는 입력 칸의 유형에 따라 다른데, `password`와 `text`에서는 글자 수(CSS `em` 단위)에 기본 값이 20이며, 나머지 유형에서는 픽셀 값입니다. CSS `width`가 `size` 특성보다 우선합니다.

</dd>

<Term>src</Term>
<dd>

`image` 유형에만 사용할 수 있습니다. 제출 버튼의 이미지로 사용할 파일의 URL을 지정합니다.

</dd>

<Term>step</Term>
<dd>

`number`, `range`, 날짜 및 시간 입력 유형에 사용할 수 있습니다. 가능한 값의 단계를 지정합니다.

지정하지 않을 경우,

- `number`와 `range` 유형에서의 기본 값은 1입니다.
- `time` 유형에서의 기본 값은 기본 값은 1초입니다.
- `date` 유형에서의 기본 값은 1일입니다.
- `week` 유형에서의 기본 값은 1주입니다.
- `month` 유형에서의 기본 값은 1월입니다.

`step` 특성의 값은 양의 실수여야 하며, 단계를 지정하지 않음을 뜻하는 키워드인 `any`도 지정할 수 있습니다.

`any` 값을 명시하지 않은 경우, 단계에 일치하지 않는 값은 유효하지 않습니다. 예를 들어 `<input type="number">`에서는 모든 정수가 유효하지만, `step`의 기본 값이 1이므로 그 외의 실수(4.2 등)는 유효하지 않습니다. 4.2를 유효한 값으로 받으려면 `step`에 `any`를 명시하거나, 0.1과 0.2처럼 4.2에 도달할 수 있는 단계를 지정하거나, `min`을 -5.2처럼 .2로 끝나는 값을 지정해야 합니다.

사용자가 입력한 값이 단계에서 벗어날 경우 유효하지 않은 입력 칸으로 취급하여 CSS `:invalid` 의사 클래스와 일치합니다.

</dd>

<Term>type</Term>
<dd>

렌더링할 컨트롤의 유형을 지정하는 문자열입니다. 예를 들어 체크박스를 생성하려면 `type` 특성에 `checkbox`를 사용합니다. 지정하지 않았거나 알 수 없는 값을 지정한 경우 `text` 유형을 보여줍니다.

가능한 유형은 위의 [`<input> 유형`](#input-유형)에서 볼 수 있습니다.

</dd>

<Term>value</Term>
<dd>

컨트롤의 값입니다. HTML 코드에서 명시한 경우 컨트롤의 초기 값으로 사용합니다. 그 후에는 JavaScript를 사용해 [`HTMLInputElement`](https://developer.mozilla.org/en-US/docs/Web/API/HTMLInputElement)의 `value` 속성으로 값을 알아내거나 바꿀 수 있습니다. `value` 특성은 명세 상 필수가 아니지만, `checkbox`, `radio`, `hidden` 유형에서는 필수 특성으로 취급해야 합니다.

</dd>

<Term>width</Term>
<dd>

`image` 유형에만 사용할 수 있습니다. 이미지의 너비를 지정합니다.

</dd>

</dl>

# 접근성 고려사항

## 레이블

`<input>`의 용도를 설명하는 레이블은 접근성 요구사항입니다. [`<label>`](./label) 요소를 사용해서, `<input>`에 입력해야 하는 값을 사용자에게 설명해주세요.

### 연관 레이블

서로 짝지어진 `<input>`과 `<label>` 요소는 스크린 리더 등 보조 기술에서 유용하게 사용합니다. `<label>` 요소의 `for` 특성을 사용하면 레이블을 입력 칸에 연결해서 보조 기술이 해당 칸을 보다 정확히 설명할 수 있도록 도와줍니다.

단순히 입력 칸 옆에 일반 텍스트를 배치하는 것으로는 충분하지 않습니다. 사용성과 접근성을 위해서는 명시적 또는 암시적인 레이블이 필요합니다.

```html variant=bad
<p>이름을 입력하세요: <input id="name" /></p>
```

위의 코드에서는 `<input>`과 "이름을 입력하세요" 텍스트 사이에 아무런 관계도 설정하지 않았으므로 접근성이 떨어집니다.

```html variant=good
<!-- 암시적 레이블 -->
<p>
  <label>이름을 입력하세요: <input id="name" /></label>
</p>

<!-- 명시적 레이블 -->
<p>
  <label for="name">이름을 입력하세요: <input id="name" /></label>
</p>
```

이름 외에도, `<label>` 요소는 마우스와 터치 스크린에서 입력 칸을 선택할 수 있는 더 넓은 "히트" 영역을 제공합니다. `<label>`을 클릭해도 연결된 `<input>`에 포커스가 가기 때문입니다.

웹 개발자로서, 사용자가 우리의 의도를 알아줄 것이라고 기대하는 것은 금물입니다. 웹에는 다양한 사람이 있으며, 각자 생각하는 방법도 그만큼 다양할 것이므로, 분명하고 뚜렷하게 나타낸 레이블 없이는 양식 역시 각자 다르게 해석할 수 있습니다.

### 플레이스홀더는 접근성이 떨어짐

[`placeholder`](#term-placeholder) 특성은 `<input>` 요소의 콘텐츠 영역이 비어있을 때 보여줄 텍스트를 지정할 수 있습니다. 그러나 플레이스홀더는 레이블이 아니므로, 레이블 대신 사용해서는 안됩니다. 플레이스홀더는 입력할 값의 형태를 보여주는 텍스트로, 설명 등을 제공하는 것이 아닙니다.

스크린 리더가 플레이스홀더를 읽지 못하는 것은 물론, 입력 칸에 값을 입력하는 순간, 또는 사전에 이미 값이 지정된 경우, 사용자는 플레이스홀더를 볼 수 없습니다. 또한 자동 번역 기능을 제공하는 브라우저에서 플레이스홀더는 번역 대상에 포함되지 않을 수 있습니다.

`placeholder` 특성을 피할 수 있다면 피하세요. 레이블이 필요하면 `<label>`을 사용해야 합니다.

## 크기

입력 칸과 같은 대화형 요소는 상호작용하기 충분한 크기의 활성화 영역을 가져야 합니다. 운동 조절 장애를 가진 사용자뿐만 아니라 터치 스크린처럼 정확하지 않은 입력 도구 사용자도 충분한 활성화 영역 크기에 도움을 받습니다. 최소 44x44의 CSS 픽셀 크기를 권고합니다.
