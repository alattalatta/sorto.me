**HTML `<link>`** 요소는 현재 문서와 외부 리소스의 관계를 명시합니다. `<link>`는 스타일 시트를 연결할 때 제일 많이 사용하지만, 사이트 아이콘("파비콘"과 홈 화면 아이콘) 연결 등 여러가지 용도로 사용할 수 있습니다.

외부 스타일 시트를 연결하려면 다음과 같이 `<link>` 요소를 [`<head>`](./head) 안에 배치하세요.

```html
<head>
  <link href="main.css" rel="stylesheet" />
</head>
```

위의 간단한 예제는 [`href`](#term-href) 특성에 스타일 시트의 경로를, [`rel`](#term-rel) 특성에 `stylesheet`을 사용합니다. `rel`은 관계(relationship)를 뜻하며 현재 문서와 연결한 아이템의 관계가 어떻게 되는지 설명하는, `<link>`의 제일 중요한 기능 중 하나라고 볼 수 있습니다. [링크 유형 참고서](https://developer.mozilla.org/en-US/docs/Web/HTML/Link_types)가 나타내듯 관계에는 많은 수가 있습니다.

그중에서도 자주 쓰이는 유형이 몇 가지 있습니다. 예컨대, 사이트와 파비콘을 연결하려면 다음과 같이 사용합니다.

```html
<link rel="icon" href="favicon.ico" />
```

아이콘에 쓰이는 `rel` 값에는 다른 것도 있는데, 주로 다양한 휴대기기 플랫폼에서 사용하는 특별한 아이콘을 지정할 때 사용합니다.

```html
<link rel="apple-touch-icon-precomposed" sizes="114x114" href="apple-icon-114.png" type="image/png" />
```

[`sizes`](#term-sizes) 특성은 아이콘 크기를 나타내고, [`type`](#term-type) 특성은 연결한 리소스의 MIME입니다. 이런 여러가지 정보는 브라우저가 가장 적합한 아이콘을 선택할 때 사용합니다.

[`media`](#term-media) 특성에 미디어 유형이나 쿼리를 지정할 수도 있는데, 그러면 해당 미디어 조건을 만족할 때만 연결된 리소스를 불러옵니다.

```html
<link href="print.css" rel="stylesheet" media="print" />
<link href="mobile.css" rel="stylesheet" media="screen and (max-width: 600px)" />
```

새로운 성능 및 보안 관련 기능도 추가됐습니다. 다음 코드로 살펴보겠습니다.

```html
<link rel="preload" href="myFont.woff2" as="font" type="font/woff2" crossorigin="anonymous" />
```

`rel`의 `preload` 값은 브라우저가 [이 리소스를 미리 불러와야 한다](https://developer.mozilla.org/en-US/docs/Web/HTML/Preloading_content)는 것을 나타내고, [`as`](#term-as) 특성은 가져오는 리소스가 어떤 리소스인지 나타냅니다. 마지막으로 [`crossorigin`](#term-crossorigin) 특성은 리소스를 CORS 요청으로 불러와야 하는지 결정합니다.

다른 참고사항도 읽어보세요.

- 링크 유형이 ["body-ok"](https://html.spec.whatwg.org/multipage/links.html#body-ok)라면 `<link>`를 [`<body>`](./body) 안에 배치할 수도 있습니다. 예를 들어 `stylesheet` 링크 유형은 body-ok며, 따라서 `<link rel="stylesheet">`은 `<body>` 내에 배치할 수 있습니다. 그러나 좋은 방법은 아니며, `<link>`는 콘텐츠와 분리해 [`<head>`](./head)에 배치하는 편이 낫습니다.
- `<link>`를 사용해 파비콘을 추가할 때, 해당 사이트가 보안 향상을 위해 콘텐츠 보안 정책(CSP)을 사용 중이라면 파비콘에도 정책이 적용됩니다. 파비콘을 불러올 수 없는 문제가 발생한다면 [`Content-Security-Policy`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Content-Security-Policy) 헤더의 [`img-src` 지시어](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Content-Security-Policy/img-src)가 접근을 막고 있나 확인해보세요.

# 특성

[전역 특성](/docs/Web/HTML/Global_attributes)을 포함합니다.

- : `as`
  - `rel="preload"` 또는 `rel="prefetch"`를 지정한 경우에만 사용합니다. `<link>`가 불러오는 콘텐츠의 유형을 나타낼 수 있으며, 요청 매칭과 올바른 콘텐츠 보안 정책 적용, 올바른 `Accept` 요청 헤더 설정에 필요합니다. 또한 `rel="preload"` 특성은 `as` 특성의 값을 요청 우선순위 결정에 참고합니다. 아래의 표는 `as`의 값과, 그에 대응하는 요소 또는 자원을 나열합니다.

    | 값         | 적용 대상                                                                                                                         |
    | ---------- | --------------------------------------------------------------------------------------------------------------------------------- |
    | `audio`    | `<audio>` 요소                                                                                                                    |
    | `document` | `<iframe>`, `<frame>` 요소                                                                                                        |
    | `embed`    | `<embed>` 요소                                                                                                                    |
    | `fetch`    | `fetch`, `XMLHttpRequest`.<br/>**참고**: 이 값을 사용하려면 `<link>`에 [`crossorigin`](#term-crossorigin) 특성도 지정해야 합니다. |
    | `font`     | CSS `@font-face`                                                                                                                  |
    | `image`    | `<img>`, `srcset` 또는 `imageset`을 지정한 `<picture>`, SVG `<image>`, CSS `*-image` 속성                                         |
    | `object`   | `<object>` 요소                                                                                                                   |
    | `script`   | `<script>` 요소, Web Worker `importScripts`                                                                                       |
    | `style`    | `<link rel="stylesheet">` 요소, CSS `@import`                                                                                     |
    | `track`    | `<track>` 요소                                                                                                                    |
    | `video`    | `<video>` 요소                                                                                                                    |
    | `worker`   | `Worker`, `SharedWorker`                                                                                                          |

- : `crossorigin`
  - CORS를 사용해 지정한 리소스를 가져올지 나타내는 열거형 특성입니다. [CORS 활성화 리소스](https://developer.mozilla.org/en-US/docs/Web/HTML/CORS_enabled_image)는 [`<canvas>`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/canvas) 요소에 사용해도 캔버스가 오염(taint)되지 않습니다. 가능한 값은 다음과 같습니다.

    - : `anonymous`
      - 자격 증명 없이 교차 출처 요청을 전송합니다. 즉, `Origin` HTTP 헤더를 쿠키, 인증서, HTTP Basic 인증 없이 전송합니다. 서버에서 `Access-Control-Allow-Origin` HTTP 헤더를 지정하지 않아서 요청 출처 사이트에 자격 증명을 보내지 않는다면 캔버스 이미지가 오염되고, 사용처가 제한됩니다.
    - : `user-credentials`
      - 자격 증명과 함께 교차 출처 요청을 전송합니다. 즉, `Origin` HTTP 헤더를 쿠키, 인증서, 또는 HTTP Basic 인증과 함께 전송합니다. 서버에서 `Access-Control-Allow-Origin` HTTP 헤더를 지정하지 않아서 요청 출처 사이트에 자격 증명을 보내지 않는다면 캔버스 이미지가 오염되고, 사용처가 제한됩니다.

    특성이 존재하지 않으면 리소스를 CORS 요청 없이(`Origin` HTTP 헤더 없이) 가져오므로, `<canvas>`가 오염됩니다. 유효하지 않은 값의 경우 `anonymous` 깂을 사용한 것으로 간주합니다.

- : `disabled`
  - `rel="stylesheet"` 전욕 특성입니다. 지정한 스타일 시트를 불러온 후 문서에 적용할지 여부를 지정합니다. HTML을 불러오는 시점에 `disabled`가 지정되어 있는 경우 스타일 시트를 불러오지 않으며, `disabled`의 값이 `false`로 바뀌거나 아예 특성이 제거되는 시점에 불러옵니다.

    DOM에서 `disabled` 특성을 지정하면 문서의 [`styleSheets`](https://developer.mozilla.org/en-US/docs/Web/API/DocumentOrShadowRoot/styleSheets) 리스트에서 스타일 시트를 제거합니다.

- : `href`
  - 연결할 리소스의 [URL](https://developer.mozilla.org/ko/docs/Glossary/URL)입니다. 절대 URL과 상대 URL 모두 지정할 수 있습니다.
- : `hreflang`
  - `href`로 연결한 링크의 인간 언어를 지정합니다. 별도의 기능은 없습니다. 전역 [`lang`](../Global_attributes#term-lang) 특성과 같은 값을
사용합니다.
- : `imagesize`
  - `as="image" rel="preload"` 전용 특성입니다. [_Sizes_](https://html.spec.whatwg.org/multipage/images.html#sizes-attributes) 특성으로, `<img>` 요소에 사용할 이미지 리소스를 미리 불러올 때 그 `sizes` 특성으로 이 값을 사용합니다.
- : `imagesrcset`
  - `as="image" rel="preload"` 전용 특성입니다. [_Srcset_](https://html.spec.whatwg.org/multipage/images.html#srcset-attributes) 특성으로, `<img>` 요소에 사용할 이미지 리소스를 미리 불러올 때 그 `srcset` 특성으로 이 값을 사용합니다.
- : `integrity` <Experimental />
  - 가져올 리소스의 해시 값을 Base64로 인코딩해 지정하면, 주어진 해시와 실제로 가져온 리소스의 해시를 브라우저가 비교해서 변형이나 조작이 있었는지 판별합니다. [하위 자원 무결성](https://developer.mozilla.org/en-US/docs/Web/Security/Subresource_Integrity) 문서를 참고하세요.
- : `media`
  - 가져온 리소스를 적용할 매체를 지정합니다. 매체 유형이나 [미디어 쿼리](https://developer.mozilla.org/en-US/docs/Web/CSS/Media_Queries) 문자열을 지정할 수 있습니다. 매체에 따라 다른 스타일 시트를 적용할 때 유용합니다.
- : `prefetch`
  - 다음 탐색 발생 시에 이 리소스가 필요할 수도 있다고 지정합니다. 사용자 에이전트는 이 리소스를 미리 불러옵니다. 실제로 리소스가 필요해지면 새로 받을 필요 없이 가지고 있는 리소스를 사용하므로 더 빠르게 응답할 수 있습니다.
- : `referrerpolicy`
  - 리소스를 가져올 때 사용할 리퍼러(referrer)를 지정합니다. 다음 값을 사용해야 합니다.

    - `no-referrer`: [`Referer`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Referer) 헤더를 전송하지 않습니다.
    - `no-referrer-when-downgrade`: TLS(HTTPS) 지원을 하지 않는 출처에 대해서는 `Referer` 헤더를 전송하지 않습니다. `referrerpolicy` 특성을 따로 지정하지 않은 경우의 기본 정책으로 사용합니다.
    - `origin`: `Referer` 헤더가 요청의 출처, 즉 스킴, 호스트, 포트를 포함합니다.
    - `origin-when-cross-origin`: 다른 출처로 요청할 땐 리퍼러 데이터를 스킴, 호스트, 포트로 제한합니다. 동일 출처로 요청할 땐 전체 경로와 쿼리 문자열까지 포함합니다.
    - `unsafe-url`: 리퍼러가 항상 출처, 경로, 쿼리 문자열을 포함합니다. 하지만 프래그먼트, 비밀번호, 사용자 이름은 포함하지 않습니다. TLS로 보호받는 리소스에서 그렇지 않은 출처로 정보를 누출할 수 있으므로 안전하지 않습니다.

- : `rel`
  - 연결하는 리소스와 현재 문서 사이의 관계를 명시합니다. 공백으로 구분한 [링크 유형 값](https://developer.mozilla.org/en-US/docs/Web/HTML/Link_types)의 목록이어야 합니다.
- : `sizes`
  - 리소스가 담고 있는 아이콘의 크기를 지정합니다. [`rel`](#term-rel)의 값이 `icon`이거나, `apple-touch-icon` 등 아이콘 관련 Apple의 비표준 값일 때만 지정해야 합니다. 가능한 값은 다음과 같습니다.

    - `any`: `image/svg+xml` 등 벡터 형식 아이콘이라서 어떠한 크기라도 될 수 있음을 나타냅니다.
    - `<너비 픽셀>x<높이 픽셀>` 또는 `<너비 픽셀>X<높이 픽셀>`의 형식으로 나타내는 아이콘 크기. 공백으로 구분한 리스트로, 다수의 값을 지정할 수 있습니다. 단 해당 리소스가 각각의 크기를 모두 가지고 있어야 합니다.

    <Callout.Root childAs="div" color="warn" label="참고">
      대부분의 아이콘 파일 형식은 단일 크기의 이미지만 저장할 수 있으므로 대부분의 `sizes` 특성 역시 하나의 값만 갖게 됩니다.
    </Callout.Root>

- : `title`
  - `<link>` 요소의 `title` 특성은 [전역 `title` 특성](../Global_attributes#term-title)과 다른 의미를 갖습니다. `<link>`의 `title`을 `rel="stylesheet"`와 함께 사용하면 [선호 스타일 시트와 대체 스타일 시트](https://developer.mozilla.org/en-US/docs/Web/CSS/Alternative_style_sheets)를 지정할 수 있습니다.
- `type`
  - 연결하는 리소스의 유형을 지정합니다. 값으로는 `text/html`, `text/css` 등 유효한 MIME 유형을 사용해야 합니다. 일반적인 사용법은 `text/css`를 지정해서 리소스가 스타일 시트임을 나타내는 것이지만, CSS가 웹에서 사용하는 유일한 스타일 시트 언어이므로 생략할 수 있습니다. 심지어 이젠 스타일 시트의 `type` 특성을 생략하는 것이 좋은 방법으로 받아들여집니다. `rel="preload"` 링크 유형과 함께 사용하면 브라우저에서 지원하는 유형의 파일만 미리 불러올 수 있도록 만들 수도 있습니다.

# 예제

## 스타일 시트 연결

```html
<link href="style.css" rel="stylesheet" />
```

## 다양한 맥락에서 사용할 아이콘 제공하기

같은 페이지에 아이콘을 다른 아이콘을 여럿 연결하면 브라우저는 [`rel`](#term-rel)과 [`sizes`](#term-sizes) 값을 참고해서 최적의 아이콘을 사용합니다.

```html
<!-- third-generation iPad with high-resolution Retina display: -->
<link rel="apple-touch-icon-precomposed" sizes="144x144" href="favicon144.png" />
<!-- iPhone with high-resolution Retina display: -->
<link rel="apple-touch-icon-precomposed" sizes="114x114" href="favicon114.png" />
<!-- first- and second-generation iPad: -->
<link rel="apple-touch-icon-precomposed" sizes="72x72" href="favicon72.png" />
<!-- non-Retina iPhone, iPod Touch, and Android 2.1+ devices: -->
<link rel="apple-touch-icon-precomposed" href="favicon57.png" />
<!-- basic favicon -->
<link rel="icon" href="favicon32.png" />
```

## 미디어 쿼리에 맞는 스타일 시트만 불러오기

요소의 [`media`](#term-media) 특성에 매체 유형이나 미디어 쿼리를 지정하면 현재 매체가 조건을 만족할 때만 리소스를 가져오게 됩니다.

```html
<link href="print.css" rel="stylesheet" media="print" />
<link href="mobile.css" rel="stylesheet" media="all" />
<link href="desktop.css" rel="stylesheet" media="screen and (min-width: 600px)" />
<link href="highres.css" rel="stylesheet" media="screen and (min-resolution: 300dpi)" />
```

## 스타일 시트 `load` 이벤트

스타일 시트를 다 불러온 시점은 `<link>` 요소에서 발생하는 `load` 이벤트를 통해 알아낼 수 있습니다. 마찬가지로, `error` 이벤트를 수신하면 스타일 시트를 불러오다가 오류가 발생하면 잡아낼 수 있습니다.

```html
<script>
  const stylesheet = document.querySelector('#stylesheet')

  myStylesheet.onload = function () {
    console.log('스타일 시트를 불러왔습니다.')
  }

  myStylesheet.onerror = function () {
    console.error('스타일 시트를 불러오지 못했습니다!')
  }
</script>

<link rel="stylesheet" href="stylesheet.css" id="stylesheet" />
```

<Callout.Root color="warn" label="참고">
  <code>load</code> 이벤트는 정확히 해당 스타일 시트 본문과 그 스타일 시트에서 가져오는 다른 콘텐츠까지 모두 불러와서
  분석을 마친 후, 콘텐츠에 적용하기 직전에 발생합니다.
</Callout.Root>
